# engcheat

このリポジトリは、モダンなWebインターフェースを通じてリスニングやリーディングのスキル向上を支援する英語学習プラットフォームです。英語テキストの読み上げに合わせて自動的にスクロールし、PDF形式の学習教材も生成することができます。

## ディレクトリ構成

- `data/`  
  英語・日本語のペアテキスト（`out.json`）と対応する音声ファイル（`out.mp3`）が格納されています。各レッスンフォルダ（例: `YYYYMMDD_article-name`）ごとに整理されています。
- `paper/`  
  `data` フォルダの内容からPDF教材（`paper.pdf`）を生成するための、RustベースのスクリプトとMakeファイルが含まれています。
- `frontend/`  
  Next.js (App Router) で構築されたWebアプリケーションです。静的HTML（Static Export）を出力し、GitHub Pagesで公開できるように設計されています。

## アプリケーションの仕組み

Webアプリケーションは、フォルダ内に展開されたJSONファイルとMP3音声を自動的に読み込みます。
各文ごとの厳密な再生タイムスタンプが元データに存在しないため、フロントエンド側でテキストの文字数の割合から現在の読み上げ位置を推測計算しています。音声の再生に合わせてテキストを自動スクロール・ハイライトし、視覚的な学習をサポートします。

### 主な機能

1. **自動スクロール＆ハイライト:** 音声の再生位置に合わせて、現在読み上げられている文がハイライト（強調表示）され、視界の中央に自動でスクロールします。
2. **自動進行＆リピート再生:** 1つの学習音声が最後まで再生されると、自動的に次のレッスンへ進みます。「リピートボタン」をONにすることで、次のレッスンへは進まず、同じ音声をループ再生させることも可能です。
3. **GitHub Pages デプロイ:** Next.jsによる静的出力により、軽量かつ高速なページロードを実現しています。

## ローカル開発環境の構築方法

Webアプリケーションをローカルで実行する場合は、以下の手順に従ってください：

1. **データの生成:**
   プロジェクトのルートディレクトリで以下のコマンドを実行します。これによりデータが処理され、`data/` と `paper.pdf` が `frontend/public/` フォルダに自動コピーされます。
   ```bash
   make generate
   ```

2. **開発サーバーの起動:**
   ```bash
   make run
   ```

3. **ブラウザで確認:**
   `http://localhost:3000` にアクセスして、記事の一覧表示や音声の再生を確認してください。

## デプロイメント

このアプリケーションは **GitHub Actions** (`.github/workflows/nextjs.yaml`) を利用して、GitHub Pagesへ自動デプロイを行っています。

リポジトリに変更がプッシュされると、以下のワークフローが自動的に走ります：
```bash
make REPONAME=<repository-name> generate deploy
```
Next.jsの静的コンパイル（`output: "export"`）が実行され、`REPONAME`（リポジトリ名）に基づいた正しいベースパス（`basePath`）があてがわれた上で、`./frontend/out` 内の出力ファイル群がGitHub Pagesとして公開されます。
